## Tech Leader Verdict

### Architecture Assessment

The 004 slice adds the auth/accounts/lease/observability layer on top of the 003 web
backend. The architectural bones are correct: an `AccountRepository` abstracts account
storage, a `LeaseService` manages session leases, an `OIDCAuth` class validates JWTs
against a JWKS endpoint, and an `OTel` setup wraps the FastAPI app. The dependency
direction is preserved — the web layer depends on interfaces, not on concrete storage
or engine internals.

However, the implementation has **critical defects** that prevent production deployment:

1. **Fail-open auth**: the production path falls back to the dev stub when `OIDC_JWKS_URI`
   is unset. This is a CRITICAL security finding — the auth layer is effectively optional.
2. **Dead code**: `AccountRepository.create_campaign` and the DB-backed campaign methods
   are never called from any route. The in-memory `CampaignRegistry` is still the live
   ownership store. The DB-backed ownership is architecture theater.
3. **Ineffective observability**: `FastAPIInstrumentor().instrument()` is called without
   the app instance, so no HTTP spans are created. The span helpers exist but are never
   wired to the play/combat routes. Metrics are declared but never emitted. The OTel
   setup is inert.
4. **PII leakage**: `span_set_error` records the full exception (message + traceback),
   and the generic error handler logs full tracebacks. This violates the no-PII rule.
5. **Lease takeover ignores `current_token`**: any session of the same account can
   force-displace the lease, defeating the purpose of lease validation.

### ADR Compliance
- 📝 **ADR-022 (OIDC fail-closed)**: Not yet implemented — the ADR is written but the code
  falls back to the dev stub. This is the CRITICAL finding.
- 📝 **ADR-023 (session lease)**: Partially implemented — `acquire`/`validate`/`release`
  exist but `takeover` ignores `current_token`; expiry uses `<` instead of `<=`.
- 📝 **ADR-024 (OTel)**: Partially implemented — the setup exists but is ineffective
  (wrong instrumentor call, no wiring, no metrics, PII in spans).
- 📝 **ADR-025 (DB-backed campaign registry)**: Not yet implemented — the repository
  exists but is dead code; play routes still use the in-memory registry.

### Cross-Report Synthesis

**Pattern 1 — Fail-open auth is the root cause.** The QA "HIGH: production falls back to
dev stub" and the Security "CRITICAL A07: hardcoded dev-token bypasses OIDC" are the same
defect. The auth layer was designed to be fail-closed but implemented as fail-open.

**Pattern 2 — Dead code masquerades as architecture.** The DB-backed `AccountRepository`
campaign methods exist but are never called. The OTel setup exists but is inert. The
lease service exists but `takeover` doesn't validate. This is a pattern: the structure is
correct but the wiring is missing.

**Pattern 3 — PII discipline is absent.** Spans record full exceptions, logs record full
tracebacks. The no-PII rule is violated at the observability layer.
