## Tech Leader Verdict

### Architecture Assessment
ADR-029 is well-reasoned and correctly implemented at the 007 layer. `effects[]` is gone, `Scene` is `narrative + choices` (scene.py:26-47), the narrator calls MCP tools directly during `agent.run()` (agent.py:128-142), and `take_turn` shrank to read → narrate → re-read → check-terminal (play.py:238-323). Principle I is satisfied by design — this is the right architecture and removes a whole class of complexity.

The problems are two bounded gaps the refactor exposed/inherited, not flaws in the 007 design:

1. **Autonomous narrator gets the full 18-tool surface, unfiltered.** The old `effects[]` path implicitly constrained the narrator to 9 safe tools via `EFFECT_TO_MCP_TOOL`; removing effects silently removed that allowlist. A single hallucination — not even an attack — can `archive_character` a live hero mid-story. Cheap fix: filter MCPToolset to read+narrate+combat+roll subset.

2. **ADR-018 (per-campaign isolation) is NOT implemented in the code.** Every `call_engine(...)` in play.py passes no `campaign_id`; app.py scopes ONE subprocess from a single env var; mcp_host.py boots by env with no `storage_factory`. ADR-029 lists this as precondition #1. It isn't met.

### ADR Compliance
- ✅ **ADR-029** (tool-use, eliminate effects[]): Compliant in code and spirit.
- ✅ **ADR-011** (PydanticAI, model-agnostic): Compliant — model injected, not hardcoded.
- ✅ **ADR-019** (fabricated-number allowlist): Correctly retired — ADR-029 makes it moot by design.
- ❌ **ADR-018** (multi-tenant per-call campaign_id): **Violation / not implemented.** Load-bearing precondition of ADR-029 unmet. Reconcile with CLAUDE.md's "006 done" claim.
- 📝 **New ADR needed**: *Narrator tool-surface allowlist + agent UsageLimits* — which of the 18 tools the autonomous narrator may invoke, and a `request_limit` cap.

### Learning Lessons Applied
- ✅ **pydantic-ai 2.0 MCPToolset `toolsets=[]` per-run**: Applied correctly — `narrate()` passes toolset per-run (agent.py:137-141).
- ✅ **FastMCP return-serialization gotchas**: No regression — `call_engine` returns plain values via `direct_call_tool`.
- ❌ No existing lesson covers **tool-surface allowlisting for an autonomous agent** — worth a Learning Lesson once the fix lands.

### Cross-Report Synthesis
The three security HIGHs collapse onto one root: A01 isolation (ADR-018 unimplemented). The other two are amplifiers whose severity is conditional on A01:
- **Prompt injection via `choice`**: in true solo play with per-account isolation, self-harm/cheating → LOW. With A01 open, reaches another tenant's hero → HIGH. Fix A01 and this drops to LOW.
- **Tool allowlist**: justified on robustness alone (hallucination archiving a live hero), independent of threat model. Fix regardless.

QA MEDIUMs calibrated:
- **Validator tautology**: real dead code, but actual impact is a UI dead-end, not a false campaign-end (`_check_terminal_state` reads engine state, not Scene). MEDIUM→LOW real impact, but it's a 2-line fix.
- **Frontend combat dead code**: incomplete ADR-029 Phase 5 scope. In-scope for 007; finish it.
