- **`update_world(changes)` — mass-assignment GUARDED.** Identical pattern to `update_character_sheet`: `unknown = set(changes) - _UPDATABLE_WORLD_FIELDS` rejects any field outside the 5-field allowlist before any mutation (server.py:193–198). No private/internal model attributes reachable.

- **No injection / unsafe deserialization introduced.** pydantic `model_validate` only; no `eval`/`exec`/`pickle`/`yaml`/format-string sinks.
- **`start_combat` validation** and **`FleeResult.hero_alive`** — pure domain logic / derived boolean. No security impact.
